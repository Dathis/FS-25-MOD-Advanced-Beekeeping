# V0.1.0 Behives and frames

## Version Goal

Version **0.1.0** is the first playable implementation of the mod.

Its goal is to deliver the **core hive gameplay loop**:

- place a beehive
- load wax frames
- insert a bee colony
- activate the hive
- manage frame states inside the hive

This version focuses only on the **minimum functional beekeeping system** required to make the mod interactive and testable.

---

# Included Features

Version **0.1.0** includes the following gameplay objects:

- **Beehive**
- **Bee Colony Box**
- **Wax Frame**
- **Box for Wax Frames**

---

# Feature Scope

## 1. Beehive

The **Beehive** is the central gameplay object of this version.

### Included functionality

- can be bought and placed by the player
- starts as an empty and non-working object
- has an interaction trigger area
- accepts:
    - wax frames
    - bee colony box
- becomes active only if:
    - frames are present
    - a bee colony is present
    - required conditions are fulfilled
- stores internal hive data:
    - bee colony presence
    - queen presence
    - number of empty frames
    - number of full frames
    - current working state

### Required states

- Empty
- HasFrames
- Active
- Inactive

### UI information

When the player looks at the hive, the following information should be shown:

- hive status
- bee colony presence
- queen presence
- empty frame count
- full frame count

---

## 2. Bee Colony Box

The **Bee Colony Box** is a portable item used to activate the hive.

### Included functionality

- can be bought in the shop
- is not a placeable building
- can be carried by the player
- contains:
    - 50,000 bees
    - 1 queen
- can be inserted into the beehive trigger area
- is consumed after successful insertion
- disappears after use

### Activation rules

The Bee Colony Box can only be accepted if:

- the hive does not already contain a colony
- the hive contains at least one frame

---

## 3. Wax Frame

The **Wax Frame** is the main reusable production item of the hive.

### Included functionality

- exists as an individual item
- can be carried manually
- has two states:
    - Empty
    - Full
- has durability
- maximum durability: 10 uses
- durability system is already implemented in the data model

### Included interactions

- can be inserted into the hive
- can be removed from the hive
- can be stored in the Box for Wax Frames

### Notes for v0.1.0

In version 0.1.0, durability must exist and be tracked, even if full extraction logic is not yet implemented.

This allows future versions to extend the system without redesigning the frame model.

---

## 4. Box for Wax Frames

The **Box for Wax Frames** is a portable container for handling frames efficiently.

### Included functionality

- can be bought in the shop
- can be carried by the player
- can store up to 12 frames
- may be bought:
    - empty
    - pre-filled with empty frames
- can be placed in the hive trigger area
- automatically transfers frames to or from the hive depending on conditions

### Transfer logic

If the box is placed in the hive trigger area:

- when the box contains valid frames and the hive has free capacity
→ frames are transferred into the hive
- when the box has free capacity and the hive contains removable frames
→ frames are transferred into the box

### v0.1.0 simplification

For MVP, transfer logic should remain simple and predictable.

Recommended rule:

- inserting frames into hive has priority when the box contains Empty Frames
- removing frames from hive has priority when the box is empty

This avoids ambiguous behavior in the first version.

# Core Gameplay Loop in v0.1.0

The playable loop for version 0.1.0 is:

1. player buys and places a Beehive
2. player buys or brings Wax Frames
3. player inserts frames into the hive manually or with a Box for Wax Frames
4. player buys a Bee Colony Box
5. player places the Bee Colony Box into the hive trigger area
6. hive becomes active
7. hive starts functioning
8. player can inspect hive state and manage frames

---

# Hive Production Scope in v0.1.0

For version **0.1.0**, the hive must support the internal logic of frame-based production.

### Minimum required logic

- active hive can convert Empty Frames into Full Frames over time
- conversion is internal to the hive
- no honey liquid output is needed yet
- no extractor is included yet
- no packaging is included yet

This version only proves that the hive production cycle works.

# Functional Objective of Version 0.1.0

Version **0.1.0** is successful if the player can:

- place a hive
- insert frames
- insert a bee colony
- activate the hive
- observe the hive working
- manage frames through direct interaction or a frame box

If these actions work reliably, the core architecture of the mod is validated.

---

# Design Priorities for v0.1.0

## 1. Stability over feature count

The first version must focus on reliable interactions, not on too many systems.

## 2. Clear trigger behavior

Every interaction with the hive should be easy to understand and consistent.

## 3. Clean state management

All included objects must already use proper internal state structures, so future versions can extend them safely.

## 4. Testability

The version should be easy to test repeatedly:

- buy items
- place hive
- load frames
- insert colony
- observe state changes

---

# Recommended Implementation Philosophy

Version 0.1.0 should be treated as a **foundation build**.

Its purpose is not to provide the full beekeeping experience yet, but to validate:

- object interactions
- storage logic
- state transitions
- core production architecture

If this version is solid, future production systems can be added on top without major rework.

---

# Summary

**Advanced Beekeeping v0.1.0** is the first playable MVP version focused on the hive itself.

It includes:

- **Beehive**
- **Bee Colony Box**
- **Wax Frame**
- **Box for Wax Frames**

Its goal is to deliver a stable and testable first gameplay loop where the player can set up and operate a beehive with frame-based logic.
